# Benchmark History Note

The public package now uses stable, versionless benchmark names.

Earlier internal milestone labels such as V4, V4.1, and V5 were useful while the
routing system was being developed, but they are not part of the public command,
manifest, script, or result-schema surface. Current public docs describe
capabilities instead:

- routing benchmark;
- operational gate;
- reference triage;
- Studio symbol matrix;
- generated-source mapping;
- transport and lifecycle probes;
- readiness audit.

This note is the only place the old milestone labels are retained, so readers
understand why older screenshots or private review notes may mention them.
